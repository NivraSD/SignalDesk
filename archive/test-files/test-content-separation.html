Content Separation Test

Testing that conversational AI responses don't appear as generated content in the Content Generator

Issue Description

Previous Bug:

When user says "I need a press release", the AI's conversational response (tips and questions) was appearing as generated content in the Content Generator feature module.

Expected Behavior:

AI's conversational responses should appear ONLY in the AI Assistant chat. Only actual generated content (with "FOR IMMEDIATE RELEASE", etc.) should appear in the Content Generator.

Test Steps

Step 1: Initial Content Request

1. Open SignalDesk at http://localhost:3000

2. Click on "Content Generator" in the feature list

3. In the AI Assistant, type: I need a press release

✅ Expected: AI gives tips and asks ONE question in the chat

✅ Expected: Content Generator shows "press-release" selected but NO content displayed

Step 2: Continue Conversation

Answer the AI's question with details about your announcement

✅ Expected: AI continues conversation, asking follow-up questions

✅ Expected: Still NO content in Content Generator (only selections)

Step 3: Generate Actual Content

Either:

  • Click "Generate Content" button in Content Generator, OR
  • Tell AI to actually generate the press release

✅ Expected: ONLY NOW should content appear in Content Generator

✅ Expected: Content should start with "FOR IMMEDIATE RELEASE"

Implementation Details

Fixed in UnifiedPlatform.js:

  • handleContentGeneratorIntegration() - Only passes content with strong indicators
  • isActualGeneratedContent() - Filters out conversational responses
  • Clears previous content when starting new content request

Fixed in ContentGeneratorModule.js:

  • Added validation in useEffect for generatedContent
  • Only accepts content with actual content markers
  • Rejects content that contains questions or conversational markers

Validation Checklist

☐ AI's tips don't appear in Content Generator

☐ AI's questions don't appear in Content Generator

☐ Content type auto-populates correctly

☐ Only actual generated content appears in the module

☐ Content fully replaces interface when generated

☐ AI panel expands when content is generated

☐ No scrolling issues

Code Verification

The following conditions prevent conversational text from appearing as content:

  • Text containing "What" and "?" is rejected
  • Text with "I can help" or "Let me" is rejected
  • Text with "tips" or "Here are" is rejected
  • Content must have strong indicators like "FOR IMMEDIATE RELEASE"
  • Content must be >400 chars with >5 lines for weak indicators
